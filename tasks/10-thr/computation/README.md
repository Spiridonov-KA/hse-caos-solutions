# Computation

После ассемблера Миша увлекся теорией чисел. Из-за забывчивости, Миша опять пропустил пролог к задаче. Вот что осталось:

На вход поступают запросы вида: `a n m` - три целых числа `uint32_t`. Для каждого такого запроса нужно посчитать `(a ^ (2 ^ n)) % m`.
По окончании расчета программа должна вывести строку вида: `<id> ans`, где `id` - номер запроса (запросы нумеруются с 1), а `ans` – ответ.
Не обязательно выводить ответы на запросы по порядку.

Миша не любит ждать – поэтому вам нужно посчитать эти запросы паралельно. В задаче не требуется паралеллизация больше чем на 4 ядра.
А еще Миша не любит лишних созданий потоков – они занимают
[много времени](https://web.archive.org/web/20250411183053/https://publikace.k.utb.cz/bitstream/handle/10563/1010147/Postprint_1010147.pdf).
Подумайте, как переиспользовать потоки для обработки запросов. Вам может пригодиться написанная ранее [очередь](../../09-fp-thr/queue).

Не забывайте про синхронизацию ввода-вывода.

Реализуйте код в файле [`solution.cpp`](./solution.cpp).
